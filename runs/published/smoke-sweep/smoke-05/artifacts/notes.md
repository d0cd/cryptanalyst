# Notes

- `code/dh.py` correctly rejects the trivial public values `0`, `1`, and `P-1`.
- The server static public key generated by the module is in the order-`Q` subgroup: `pow(STATIC_PUB, Q, P) == 1`.
- The peer public value validation only checks the integer range `2 <= y <= P-2`; it does not check `pow(y, Q, P) == 1`. For this safe-prime group with `G = 2`, that admits values outside the subgroup generated by `G`.
- A comparison with Python `cryptography` showed that its DH exchange rejects `P-2` for these parameters with `ValueError: Error computing shared key`, while accepting `2`.
